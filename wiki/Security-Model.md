# Security Model

Schnipsel renders untrusted HTML — first from arbitrary web pages, later from other users. The security model is designed so that no single failure can lead to code execution or data exfiltration. Each layer is independent.

---

## Threat model

**Own clips** — serialised from a page the user chose to visit. The HTML is as trusted as any web page the user browses. Still sanitised to avoid surprises when the clip is rendered outside its original context (different origin, different CSP).

**Friend clips** — arrive as a bundle signed and encrypted by a friend, but the friend could themselves be compromised, or the bundle content could have been manipulated before the friend sent it. Every byte of a friend clip is treated as fully untrusted.

**Hostile peers / leaked invite codes** — the friend system is designed to be mutual; a leaked code cannot silently add a stranger. But once a friendship exists, the friend's shared content must be treated as potentially hostile.

The goal: a compromised or malicious friend's clip must not be able to execute JavaScript, phone home to a remote server, or pollute the local search index or data model.

---

## Defence layers

Each layer is independent. The failure of any single one still leaves the others intact.

### 1. DOMPurify

All clip HTML passes through `DOMPurify.sanitize()` (vendored at `vendor/dompurify.js`, pinned to version 3.2.4, audited before vendoring) before it is stored or rendered.

Configuration:
```js
DOMPurify.sanitize(html, {
  FORBID_TAGS: ["script", "iframe", "object", "embed", "form", "base", "meta", "link", "noscript", "template"],
  FORBID_ATTR: ["ping", "formaction", "target"],
  ALLOW_DATA_ATTR: true,          // preserve data-schnipsel-* markers
  USE_PROFILES: { html: true, svg: false, svgFilters: false, mathMl: false },
})
```

SVG and MathML profiles are disabled to prevent filter-based side-channels and SVG-embedded script. The `FORBID_TAGS` list covers the most common vectors; DOMPurify would block most of them by default anyway — this is belt-and-suspenders.

**Vendor audit:** before vendoring, `dompurify.js` was checked for absence of `fetch`, `XMLHttpRequest`, `sendBeacon`, `WebSocket`, `new Image`, `eval`, and `new Function`. The SHA-256 hash is pinned in `vendor/README.md`. To re-verify:

```bash
sha256sum vendor/dompurify.js
grep -nE "fetch|XMLHttpRequest|sendBeacon|WebSocket|new Image|eval\(|new Function" vendor/dompurify.js
```

**Fail-closed:** if DOMPurify is not loaded for any reason, `sanitize.clip()` returns an empty string rather than rendering raw HTML:

```js
if (typeof DOMPurify === "undefined") {
  console.error("Schnipsel: DOMPurify not loaded — refusing to render clip HTML.");
  return "";
}
```

### 2. Remote-reference stripping (friend clips only)

DOMPurify removes active content but leaves resource references (`src`, `srcset`, `poster`, etc.) intact. For own clips this is fine — the user's browser would load those resources when visiting the page normally. For friend clips, loading a remote resource would reveal the user's IP address and the fact that they received and viewed that clip.

`sanitize.stripRemoteRefs()` removes all resource-loading attributes whose value is not a `data:` URI:

```js
const REMOTE_ATTRS = ["src", "srcset", "poster", "background", "data", "href"];
// For each element: remove any of these if the value doesn't start with data:
// Also: strip url(...) from inline styles and <style> blocks, keeping only data: URIs
```

After this pass, a friend clip that is rendered cannot make any outbound network request, even with scripts already blocked by other layers.

### 3. iframe sandboxing

Every clip is rendered inside an `<iframe sandbox>`. The sandbox attribute is managed by `sanitize.IFRAME_SANDBOX` and `sanitize.IFRAME_SANDBOX_POPUP`:

```js
IFRAME_SANDBOX: ""
IFRAME_SANDBOX_POPUP: "allow-popups allow-popups-to-escape-sandbox"
```

**`allow-scripts` is never included.** This is a non-negotiable invariant — the code comment explicitly notes it and contributors should treat any PR that adds `allow-scripts` as a security regression.

**`allow-same-origin` is never included.** Without it, the framed document gets an opaque origin (`null`), so it cannot read cookies, localStorage, or make credentialled requests even if a sandbox escape were somehow possible.

`IFRAME_SANDBOX_POPUP` is used in the collage canvas and export so that links inside clip cards open in a new tab (`allow-popups`). `allow-popups-to-escape-sandbox` lets those new tabs operate normally (not themselves sandboxed).

### 4. Content Security Policy in iframe documents

Every clip's `srcdoc` document begins with a `<meta>` CSP tag generated by `sanitize.cspMeta(allowRemote)`:

```
default-src 'none'
img-src data: [https: http: if own clip]
media-src data: [https: http: if own clip]
font-src data: [https: http: if own clip]
style-src 'unsafe-inline'
script-src 'none'
object-src 'none'
base-uri 'none'
form-action 'none'
```

`script-src 'none'` means no JavaScript, ever, regardless of the sandbox attribute. `unsafe-inline` is needed for styles because clips depend heavily on inlined `getComputedStyle` output.

For friend clips, `allowRemote = false` drops the `https: http:` exceptions, so images, fonts, and media must be data URIs — in practice only the ones embedded at clip-time will load.

### 5. Background message sender gate

The background script verifies the sender of every incoming message. Content scripts run inside arbitrary web pages and must be treated as untrusted:

```js
function isTrustedSender(sender) {
  return !!(
    sender &&
    sender.id === browser.runtime.id &&
    typeof sender.url === "string" &&
    sender.url.startsWith(browser.runtime.getURL(""))
  );
}
```

Only messages from extension pages (`moz-extension://<id>/…`) can trigger sensitive operations. Content scripts may only send `CLIP_CREATED`.

### 6. Cryptographic bundle verification (friend shares)

The 8-step ingest pipeline in `transport.js` is an additional layer that applies before any friend clip reaches storage or DOMPurify:

1. Size + schema validation (reject oversized or malformed codes)
2. Sender is a known friend with matching stored keys
3. Fingerprint = SHA-256 of the embedded signing key (binds identity to key)
4. ECDSA signature verifies (authenticity + integrity of the envelope)
5. Bundle is addressed to us (we can unwrap the content key)
6. Replay check (bag version must be newer than what we already have)
7. Per-clip content hash check: `SHA-256(html + sourceUrl)` must equal the claimed clip ID
8. DOMPurify + remote-ref strip + size caps

Steps 1–7 happen before the HTML is ever parsed. A bundle that fails any step is rejected entirely; nothing is stored.

### 7. Search index hardening

The inverted index uses token strings as object property keys. Friend-sourced tokens are untrusted, so:

- `UNSAFE_KEYS = new Set(["__proto__", "prototype", "constructor"])` — these are never written as keys
- `buildMap` (used to build a friend's index) uses `Object.create(null)` — no prototype chain to pollute
- Per-friend indices are never merged into the local index

### 8. Avatar hardening

- Only PNG, JPEG, and WEBP data URIs are accepted (SVG is rejected — SVG can carry inline `<script>`)
- The UI re-encodes uploaded images through an HTML Canvas before saving, stripping any metadata and normalising the format
- Size cap: 64 KB encoded (covers a 128×128 pixel PNG)

### 9. Extension manifest CSP

```json
"content_security_policy": "script-src 'self'; object-src 'none'"
```

Extension pages cannot load or run JavaScript from any external source. Plugins and Java applets are blocked.

## Summary table

| Threat | Mitigated by |
|---|---|
| Script in clip HTML | DOMPurify + `script-src 'none'` CSP + `sandbox` without `allow-scripts` |
| Tracking via remote resources (friend clips) | Remote-ref stripping + `img/font/media-src data:` CSP |
| Compromised content script sending privileged messages | `isTrustedSender` gate in background |
| Forged friend identity | Fingerprint = hash of signing key; signature verification |
| Tampered clip content | Per-clip `SHA-256(html + sourceUrl)` integrity check |
| Replay of an old bag bundle | Monotonic `version` counter on bags |
| Prototype pollution via friend tokens | `UNSAFE_KEYS` + `Object.create(null)` in `buildMap` |
| Malicious friend avatar (SVG with script) | SVG rejected; canvas re-encode; size cap |
| Disk exhaustion via friend content | Per-friend quota (2000 clips) + per-bundle cap (1000 clips) |
| External JS in extension pages | Manifest `script-src 'self'` |
